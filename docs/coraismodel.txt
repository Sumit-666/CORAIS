CORAIS Design Philosophy
CORAIS is not a wrapper around OpenAI. It is an intelligent routing engine that selects the best AI model for each task, learns from outcomes, manages costs dynamically, and connects 5 AI workforce agents into a unified intelligent system.
1.1 Core Principles
Principle	Implementation
Never single-model dependent	Model Registry with 6+ models across 4 providers. Any model can be swapped, added, or removed without code changes.
CRI accuracy never degrades	CRI-affecting tasks (resume parse, code eval, interview eval) always get the best available model regardless of budget pressure.
Security never degrades	Fraud detection always gets premium model for S3+ severity. Trust Protocol is never budget-throttled.
Cost-optimized by default	Decision Gate resolves 62% of requests at $0 (cache + rules + templates + scoring). Only 38% hit paid models.
Learn from outcomes	Every AI call tracks quality signals. Monthly analysis shows which models perform best per task. Routing adapts.
Fail gracefully	If primary model is down, automatic failover to ranked fallback. If all models down, deterministic fallback where possible, queued retry for the rest.
Multi-language native	Indian language tasks route through Bhashini (cheaper + better). English tasks route through OpenAI/Claude. Language detection is the first routing decision.
Transparent to agents	AI Workforce agents call CORAIS with a prompt_type. They never specify which model to use. CORAIS decides. Agent code stays clean.

2. Intelligence Layer 1: Model Registry
The Model Registry is CORAIS's knowledge of what AI models are available, what each is good at, what each costs, and whether each is currently healthy.

model_registry

2.3 Health Monitoring
MODEL HEALTH CHECK (runs every 60 seconds per active model):

async function checkModelHealth(modelId: string) {
  const model = await getModel(modelId);
  const adapter = getAdapter(model.provider);
  
  try {
    const start = Date.now();
    await adapter.ping(model);  // lightweight test call
    const latency = Date.now() - start;
    
    await updateModel(modelId, {
      health_status: latency > model.p99_latency_ms * 2 ? 'degraded' : 'healthy',
      last_health_check: new Date(),
      avg_latency_ms: rollingAvg(model.avg_latency_ms, latency)
    });
  } catch (error) {
    await updateModel(modelId, {
      health_status: 'down',
      last_health_check: new Date()
    });
    // Alert: model ${modelId} is DOWN
    emitEvent('model_health_changed', {modelId, status: 'down'});
  }
}

4. Intelligence Layer 3: Budget-Aware Degradation
CORAIS manages a daily $50 budget ceiling. Instead of a hard wall at $50, the system makes intelligent trade-offs as spending increases.
4.1 Budget Tiers
Tier	Spend Range	Status	Behavior
NORMAL	$0 - $30 (0-60%)	Green	All models available. Premium (GPT-4o) used for Hard code evals and S3+ fraud. Full content generation. No restrictions.
CONSERVE	$30 - $40 (60-80%)	Amber	Non-critical tasks downgrade: content max_tokens 500→300, cover letter 1/day/candidate, content daily limit 10→5. CRI-affecting and security tasks unchanged.
CRITICAL	$40 - $47.50 (80-95%)	Red	Content generation PAUSED. Cover letter PAUSED. Only CRI-affecting tasks (resume parse, code eval, interview) and security tasks (fraud) process. Everything else queued for next day.
STOPPED	$47.50 - $50 (95-100%)	Dark Red	Only cache-served and deterministic requests. CRI-affecting tasks still process (they're the product). Super Admin alerted. Can override ceiling temporarily.
OVERRIDE	Above $50	Manual	Super Admin explicitly raised ceiling. Logged as admin_action. Auto-reverts at midnight.
4.2 Task Priority Classification
Task Type	Priority	Budget Behavior	Rationale
CRI Computation	CRITICAL (but $0)	Never affected	Pure math. No LLM. Always runs.
MCQ Grading	CRITICAL (but $0)	Never affected	Deterministic. No LLM. Always runs.
Gold Standard Check	CRITICAL (but $0)	Never affected	Deterministic. No LLM. Always runs.
Resume Parse	CRITICAL	Never degraded	CRI-affecting. The product IS the score.
Code Evaluation	CRITICAL	Never degraded. Hard eval may downgrade from GPT-4o to GPT-4o-mini at CONSERVE.	CRI-affecting. Accuracy matters. Hard eval cost trade-off acceptable at CONSERVE.
Interview Evaluation	CRITICAL	Never degraded	CRI-affecting. Communication scoring.
Fraud Triage	CRITICAL	Never degraded. S3+ always gets premium model.	Security. Never compromised.

Content Generation	NORMAL	Shortened at CONSERVE, paused at CRITICAL	Nice-to-have. Candidate can wait until tomorrow.
Cover Letter	NORMAL	Limited at CONSERVE, paused at CRITICAL	Nice-to-have. Can wait.

4.3 Budget Manager Implementation
class BudgetManager {
  private dailyCeiling: number = 50.00;  // USD
  private overrideCeiling: number | null = null;
  
  async getCurrentSpend(): Promise<number> {
    return await db.query(
      'SELECT COALESCE(SUM(cost_usd), 0) FROM routing_logs
       WHERE created_at >= CURRENT_DATE'
    );
  }
  
  async getBudgetTier(): Promise<BudgetTier> {
    const spend = await this.getCurrentSpend();
    const ceiling = this.overrideCeiling ?? this.dailyCeiling;
    const pct = spend / ceiling;
    if (pct < 0.60) return 'NORMAL';
    if (pct < 0.80) return 'CONSERVE';
    if (pct < 0.95) return 'CRITICAL';
    return 'STOPPED';
  }
  
  async canProcess(taskPriority: string): Promise<boolean> {
    const tier = await this.getBudgetTier();
    if (taskPriority === 'CRITICAL') return true;  // ALWAYS
    if (tier === 'STOPPED') return false;
    if (tier === 'CRITICAL' && taskPriority === 'NORMAL') return false;
    return true;
  }
  
  async getMaxTokens(promptType: string): Promise<number> {
    const tier = await this.getBudgetTier();
    const defaults = { content_gen: 500, cover_letter: 800 };
    if (tier === 'CONSERVE') return Math.floor(defaults[promptType] * 0.6);
    return defaults[promptType];
  }
}

6. Enhanced Decision Gate v3.5
The complete 8-stage routing pipeline that every AI request passes through.
6.1 Stage-by-Stage Flow
Stage 1: Cache Check
•	Redis exact-match: key = SHA-256(prompt_type + JSON.stringify(input))
•	TTL: 24 hours for resume parse, JD parse. 1 hour for content gen (more variability desired).
•	Cache bypassed for: fraud triage (always fresh), interview eval (unique per session).
IF cache HIT → return cached result. Cost: $0. Latency: <5ms.
Stage 2: Deterministic Rules
•	CRI computation: pure math (D0 formulas). No LLM needed. $0.
•	TQS computation: gold standard comparison. Deterministic. $0.
•	ESS computation: evidence strength score formula. $0.
•	MCQ grading: answer key comparison. $0.
IF rule match → compute locally. Cost: $0. Latency: <50ms.
Stage 3: Template Fill
•	WhatsApp templates: variable substitution ({name}, {score}, {job_title}). $0.
•	Email templates: same pattern. $0.
IF template match → fill variables. Cost: $0. Latency: <10ms.
Stage 4: Budget Check (NEW)
•	BudgetManager.getBudgetTier() → NORMAL / CONSERVE / CRITICAL / STOPPED
•	BudgetManager.canProcess(taskPriority) → true / false
•	If cannot process: return 429 with {reason, retry_after, budget_tier}
•	If CONSERVE: adjust max_tokens for non-critical tasks
IF budget_tier === 'STOPPED' AND task.priority !== 'CRITICAL':
  → return 429 {reason: 'Daily budget reached. CRI tasks still processing.',
     retry_after: secondsUntilMidnight()}
Stage 5: Model Selection (NEW — The Intelligence)
•	Lookup task_model_routing for this prompt_type
•	Filter by: language, difficulty, budget_tier, user_tier
•	Sort candidates by priority
•	For each candidate (in priority order):
◦	Check model_registry.health_status === 'healthy'
◦	Check model_registry.is_active === true
◦	Check rate limit not exceeded
◦	If all pass: SELECT this model
◦	If fails: try next priority
•	If multi-step pipeline: validate ALL models in pipeline are healthy
•	If no model available: return 503 {retry_after: 30}
async function selectModel(promptType, conditions) {
  const routes = await getRoutes(promptType, conditions);
  // routes sorted by priority ASC
  for (const route of routes) {
    if (route.pipeline_steps) {
      // Multi-model pipeline: ALL models must be healthy
      const allHealthy = await Promise.all(
        route.pipeline_steps.map(s => isModelHealthy(s.model_id))
      );
      if (allHealthy.every(h => h)) return route;
    } else {
      if (await isModelHealthy(route.model_id)) return route;
    }
  }
  return null;  // no model available
}
Stage 6: Execute
•	Single-model: call adapter directly
•	Multi-model pipeline: execute steps sequentially, pass output of step N as input to step N+1
•	Each step independently timed and cost-tracked
•	If any step fails mid-pipeline: rollback (discard partial results), try fallback route
async function executePipeline(route, input) {
  const steps = route.pipeline_steps || [{model_id: route.model_id}];
  let currentInput = input;
  const stepLogs = [];
  
  for (const step of steps) {
    const adapter = getAdapter(step.provider);
    const start = Date.now();
    try {
      const result = await adapter.execute(step.model_id, step.prompt, currentInput);
      stepLogs.push({model: step.model_id, latency: Date.now()-start, success: true});
      currentInput = result;  // output becomes next step's input
    } catch (error) {
      stepLogs.push({model: step.model_id, latency: Date.now()-start, success: false});
      throw new PipelineStepError(step.model_id, error, stepLogs);
    }
  }
  return {result: currentInput, stepLogs};
}
Stage 7: Quality Signal Setup (NEW)
•	Register quality tracking hooks based on prompt_type
•	resume_parse: set hook to track candidate field edits within 24 hours
•	code_eval: set hook to track HITL override within 48 hours
•	content_gen: set hook to track share/discard within 24 hours
•	Quality signal written to model_performance_log when hook fires
Stage 8: Cache Write
•	If response is cacheable: write to Redis with appropriate TTL
•	Cache key includes: prompt_type + input hash (not model_id — so cache works across model switches)
Entire 8-stage pipeline completes in < 15 seconds for any request type.
 
7. Multi-Model Pipelines
Some tasks require multiple models in sequence. CORAIS orchestrates these as atomic pipelines — either all steps succeed or the entire pipeline retries on a fallback route.

8. Agent-CORAIS Integration
Every AI Workforce agent calls CORAIS through a unified interface. Agents never specify which model to use — CORAIS decides based on task type, language, difficulty, and budget.
8.1 Agent Call Interface
// Every agent calls CORAIS the same way:
interface CORAISRequest {
  prompt_type: PromptType;  // 'resume_parse', 'code_eval', etc.
  input: any;  // the data to process
  context: {
    candidate_id?: string;
    tenant_id?: string;
    language?: string;  // 'en', 'hi', 'te', etc.
    difficulty?: 'Easy' | 'Medium' | 'Hard';
    is_retry?: boolean;
    correlation_id: string;  // for tracing event chains
  };
}

interface CORAISResponse {
  result: any;  // the AI output
  metadata: {
    model_used: string;  // which model(s) actually processed this
    pipeline_steps?: string[];  // for multi-model pipelines
    total_cost_usd: number;
    total_latency_ms: number;
    cache_hit: boolean;
    budget_tier: BudgetTier;
    quality_tracking_id: string;  // for quality signal linkage
  };
}

// Agent code stays clean:
// Agent 1 (Resume Parser):
const result = await corais.process({
  prompt_type: 'resume_parse',
  input: { text: resumeText },
  context: { candidate_id, language: detectedLanguage }
});
// Agent doesn't know or care if GPT-4o-mini, Bhashini pipeline,
// or Claude processed it. Agent gets structured skills back.
8.2 Agent-Specific CORAIS Behavior
Agent	Prompt Types Used	CORAIS Behavior	Special Rules
Agent 1: Resume Parser	resume_parse	Language detection → EN route or Hindi pipeline. Cache aggressively (same resume = same skills).	HITL at confidence < 0.6. Quality tracked via candidate edits.
Agent 2: Challenge Evaluator	code_eval	Difficulty-aware: Easy/Med → gpt-4o-mini, Hard → gpt-4o. MCQ at $0 (no CORAIS call).	HITL at integrity < 0.5. Quality tracked via admin override.
Agent 3: Task Quality Validator	(none — deterministic)	Does NOT call CORAIS. Gold standard comparison is pure logic. $0.	N/A. Agent 3 never touches the AI gateway.
Agent 4: Content Generator	content_gen, cover_letter	Language-aware pipeline. First to budget-degrade. Quality tracked via candidate share/discard.	Daily limit enforced (10 free tier). Max_tokens reduced at CONSERVE.
Agent 5: Fraud Detector	fraud_triage, code_logic_hash	Severity-aware: S0-S2 → gpt-4o-mini. S3+ → gpt-4o (premium). NEVER budget-degraded.	Quality tracked via admin agreement with AI recommendation.
8.3 Event Flow: Agent → CORAIS → Agent
EXAMPLE: Student uploads Hindi resume

1. Frontend: POST /v1/candidates/resume (Hindi PDF)
2. Agent 1 (Resume Parser): detects language = HI
3. Agent 1 calls CORAIS: {prompt_type: 'resume_parse', language: 'hi'}
4. CORAIS Decision Gate:
   Stage 1: Cache MISS (first upload)
   Stage 4: Budget = NORMAL
   Stage 5: Select pipeline: bhashini-nmt → gpt-4o-mini
   Stage 6: Execute pipeline (translate → parse)
   Stage 7: Set quality hook (track edits in 24h)
   Stage 8: Cache write
5. CORAIS returns: {skills: ['Python','SQL','Excel'], ...}
6. Agent 1 emits EVENT: resume_parsed
7. CRI Engine subscribes: recomputes F_REL + F_READY
8. CRI Engine emits: cri_updated {delta: +12}
9. WhatsApp Agent subscribes: sends T4 score_update (in Hindi via Bhashini)
10. College Dashboard subscribes: campus CRI recalculates
11. Fraud Detector subscribes: checks for duplicate submission

TOTAL: 11 actions from 1 resume upload.
CORAIS involvement: Steps 3-5 (model selection + execution).
Agent communication: Steps 6-11 (event-driven, async).
All traced via single correlation_id.

11. Adapter Specifications
Each provider is integrated via a single adapter file implementing a common interface. Adding a new provider = one new file + model_registry rows.
11.1 Common Interface
interface ModelAdapter {
  provider: string;
  
  // Health check (lightweight, < 1 second)
  ping(model: ModelRegistryEntry): Promise<boolean>;
  
  // Execute a prompt
  execute(modelId: string, prompt: string, input: any,
          options?: {max_tokens?: number, temperature?: number}
  ): Promise<{output: any, usage: {input_tokens: number,
             output_tokens: number}, latency_ms: number}>;
  
  // Calculate cost from usage
  calculateCost(modelId: string, usage: TokenUsage): number;
}
11.2 Adapter Files
File	Models Served	External Dependency	Build Milestone
