CORAIS — Complete Architecture & Concept Summary
================================================

1. What is CORAIS?
-------------------
CORAIS is the central AI orchestration brain of the platform.

It intelligently manages:
- AI workflows
- model selection
- memory
- event-driven automation
- multi-agent coordination
- reasoning
- governance
- human approvals

Simple Definition:
CORAIS is an AI operating system for the Jobazy ecosystem.

It decides:
- which model to use
- which workflow to run
- which agent should handle the task
- when humans should approve actions
- how AI services communicate together

------------------------------------------------------------

2. Difference Between API Gateway and CORAIS
--------------------------------------------

API Gateway:
- infrastructure layer
- authentication
- routing
- tenant resolution
- rate limiting
- security

CORAIS:
- AI intelligence layer
- orchestration
- reasoning
- workflow automation
- model routing
- AI decision engine

Simple Analogy:
API Gateway = Security Guard
CORAIS = Brain

Flow:
Frontend
   ↓
API Gateway
   ↓
Backend Services
   ↓
CORAIS AI Brain

5. Kafka in CORAIS
------------------

Kafka is the event-driven communication system.

Purpose:
- async communication
- decoupled architecture
- scalable workflows

Example Flow:

Recruiter creates job
        ↓
JOB_CREATED event
        ↓
Kafka Topic
        ↓
Consumers React

Consumers:
- JD Parser
- Notification Service
- Analytics
- Recommendation Engine

Producer:
Service sending event

Consumer:
Service listening to event

Topic:
Event channel/category

------------------------------------------------------------

6. Memory in CORAIS
-------------------

Memory stores user context and workflow history.

Examples:
- candidate skills
- recruiter preferences
- interview history
- application history
- weak areas
- preferred roles

Types:
- short-term memory
- long-term memory

Technologies:
- Redis
- PostgreSQL
- Vector DB

------------------------------------------------------------

7. CRI Rules Engine
-------------------

CRI = Career Readiness Index

Purpose:
- measure candidate readiness
- ranking
- scoring

Example Factors:
- skills
- projects
- GitHub activity
- communication
- interview performance

Example Rule:

if github_verified:
    score += 10

if communication_score < 40:
    score -= 15

------------------------------------------------------------

8. HILT Layer
-------------

HILT = Human-in-the-Loop Governance

Purpose:
- approval workflows
- enterprise safety
- auditability

Examples:

T0:
AI suggestions only

T1:
human confirmation required

T2:
critical decisions

T3:
admin-only operations

Example:
AI shortlist generated
      ↓
Recruiter approval required
      ↓
Workflow continues

------------------------------------------------------------

9. Agentic RAG in CORAIS
------------------------

CORAIS uses Agentic RAG internally.

RAG:
Retrieve data + generate response.

Agentic RAG:
AI agents reason through retrieval workflows.

Example:
- retrieve candidates
- analyze CRI
- check GitHub
- rank candidates
- generate reasoning

But:
CORAIS is larger than just Agentic RAG.

CORAIS also includes:
- orchestration
- Kafka events
- governance
- model routing
- memory
- workflows

------------------------------------------------------------

10. LangGraph and CORAIS
------------------------

LangGraph is the closest framework to CORAIS architecture.

LangGraph provides:
- workflow graphs
- memory
- multi-agent orchestration
- branching
- state management
- human-in-loop workflows

Core Concepts:
- Nodes = agents/tasks
- Edges = workflow flow
- State = shared memory

Example Workflow:
JD Parser
   ↓
Retriever
   ↓
CRI Scorer
   ↓
Notification Agent
